# Figma Build Direction

## File Architecture

Pages:

- `00 Foundations`
- `01 Components`
- `02 Patterns`
- `03 QA and Examples`

## Foundations Page

Include:

- Color primitives and semantic mapping
- Typography roles with examples
- Spacing, radius, elevation, and motion references
- Accessibility contrast matrix (key text and controls)

## Components Page

Sections:

- Navigation
- Actions
- Data cards
- Inputs
- Feedback

For each component include:

- Anatomy diagram
- Variants and sizes
- State matrix
- Do and do-not examples

## Patterns Page

Create blueprint frames for:

- Home dashboard
- Goals grid
- Purchase flow
- Receipt success flow

Each pattern must show:

- Spacing callouts
- Component references
- Content density limits

## QA Page

Include:

- Gold accent ratio checks
- Contrast checks
- Touch target checks
- State completeness checks

## Component Naming

Use the same naming contract from component inventory:
`Domain/Component/Variant/Size/State`
